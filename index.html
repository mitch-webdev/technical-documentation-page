Boot Process

GRUB - Grand Unified Bootloader

The boot loader is responsible for starting the operating system at startup. If you were to have different operating systems installed on a machine, this is where you would choose which operating system to start. During boot process, a bunch of messages will be displayed on the screen and will allow you to interrupt the boot process if needed. You can also tell the boat loader to pass different boot option to the operating system.


Initial RAM Disk

Abbreviated initrd, this is a temporary filesystem that is loaded into memory from the disk when the system boots. This filesystem can contain helpers that perform hardware detection and load the necessary modules like drivers.


Kernel Ring Buffer

The Kernel Ring Buffer contains messages from the Linux kernel. A ring buffer is a data structure that is always the same size. Once the buffer is completely full, all the messages will be discarded when new message arrive.

To see the content of the content of the Kernel Ring Buffer, use this command:

  • dmesg

Messages location:

  • /var/log/dmseg

In the /var/log/dmesg directory, you will be able to look at the log files that fly by during Linux boot process.


To wrap-up this boot process section, here are a few useful reboot and shutdown commands you can enter in your Linux terminal.

  • shutdown -r 22:00 "rebooting!" - Shuts down the system at 10:00 PM with the message rebooting!
  • shutdown -r +5 "rebooting soon!" - Shuts downs the system in 5 minutes with the message rebooting soon!
  • shutdown -r now - Shuts down the system right now without message
  • reboot - Reboots the system
Disk Management

Partitions

When a disk or storage device is partitioned, it is divided into different parts. Partitioning a disk allows you to divide different sectons on the disk to use them for different purposes. For example, we could create 4 partitions and allocate the first part for the operating system, a second partition would then be used for all application data, the third for the user data, and the last partition could be used for swap space. Of course, This is an example and the partitions can be allocated in different way. Having separate partitions is a way to prevent a part of the system to affect another part of the system. A partition dedicated for user home directories for example would prevent a user from filling the entire disk and interfer with the normal operation of the operating system.

This command will show you information on the disk space being used

  • df -h - Will display information about disk space used on all disks from that system

Mount Points

A mount point is simply a directory that is used to access the data on a particular partition.

  • There will always be at least a /(slash) as a mount point
  • Any additional partitons will be mounted on mount points below / in the directory tree.
  • If you allocated a partition for user home directories, that partition would be mounted at /home.

Here is how to mount partitions to a directory

  • mount /dev/sdb3 /home - You will mount the partition located at /dev/sdb3 to the directory /home

To make sure that mount point remains after reboot, it needs to be entered in /etc/fstab file.


To unmount a partition, simply use this command:

  • umount /home - Here we used the mount point itself
  • umount /dev/sdb3 - And there we used the device path

fdisk

The fdisk utility is used to create and modify partitions on a disk.

Example of an fdisk command:

  • fdisk /path/to/disk - Will open the fdisk utility to manage partitions on that disk

Shell Scripting

A script is a file that can contain multiple commands. Any command that is accepted in the command line can be added into a script and will be executed by the interpreter. This is typically used to automate tasks.

This is what a script looks like:

                        
  #!/bin/bash
  echo "Bash Scripting is a lot of fun!"


Shell scripts can of course take variables

  • Variables are case-sensitive
  • By convention, variables are uppercase

                        
  VARIABLE_NAME="Value"


Example of variable usage in shell script:

                    
  #!/bin/bash
  MY_SCRIPT="awesome!"
  echo "I think that shell scripting is $MY_SCRIPT"


Output - I think that shell scripting is awesome!

LVM - Logical Volume Manager

In Linux, Logical Volume Manager (LVM) is a device mapper framework that provides logical volume management for the Linux kernel. Most modern Linux distributions are LVM-aware to the point of being able to have their root file systems on a logical volume.

Why would you use LVM?

LVM allows you to create file systems that extend across multliple storage devices. You can aggregate multiple storage devices into a single logical volume.

  • Easily resize storage while still online
  • Data can be migrated easily from one device to another while online
  • Humand-readable device names of your choice can be used

Logical Volume Creation Process

  1. Create one or more physical volumes
  2. Create a volume group from those or more physical volumes
  3. Create one or more logical volumes from the volume group
User Management

Linux is a multi-user operating system. Multiple accounts can exist on a system and can be used at the same time.

Each account includes the following:

  • Username (or Login ID)
  • UID (User ID) - This is a unique number
  • Default group
  • Comments
  • Shell
  • Home Directory

All the information above is stored in the /etc/passwd file.

Here is an extract:

michael:x:1000:1000:,,,:/home/michael:/bin/bash


Breakdown of the passwd file
Username Password User ID Default group ID
michael x 1000 1000
Comments Home Directory Shell
,,, /home/michael /bin/bash

Let's create a user. To create a new user, enter the following command:

useradd -c "John Doe" -m -s /bin/bash john


Command syntax
Command Add a comment Create the home directory
useradd -c "John Doe" -m
The path to user's shell Desired username
-s /bin/bash john

Now let's set a password to that user.

passwd john

You will be prompted to enter the desired password for that user. Once entered, the message "password updated successfully will be displayed".

Networking

IP Address

To show the current IP address or all the IP addresses in use on a Linux system, enter the following command:

  • ip address, ip a or ip addr - This will display all IP addresses used on the system

Hostname

The following commands will display or change the current device hostname. Let's how to do that.

  • hostname - Displays the current hostname of system
  • hostname desiredhostname - Will temporarily set the desired hostname until next reboot
  • hostname 'desiredhostname' > /etc/hostname - Will permantently define desired hostname to the current device

DNS

The following tools will help you lookup or resolve DNS names or IP addresses.

  • host www.google.com or 172.217.168.46 - Will display DNS information about the domain name google.com or the IP address 172.217.168.46
  • dig www.google.com - Will also display DNS information like DNS Records of the argument given
